# Signal - Decentralized Disaster Relief Network

**Signal** is an offline-first, decentralized mesh network application designed for disaster relief situations where traditional cellular and internet infrastructure has collapsed. 

In a crisis, communication is the difference between life and death. Signal allows victims to broadcast their status (SOS, Needs, Offers, Alerts) and sync this data directly from phone-to-phone (or laptop-to-phone) without relying on a central server.

---

## 🚀 Features

- **Offline-First Architecture**: Built using Progressive Web App (PWA) technologies, meaning it works seamlessly even with zero internet connection.
- **Decentralized Mesh Syncing**: Data is stored locally on each device using IndexedDB. Devices sync data with each other directly using WebRTC (Peer-to-Peer) or via a local Relay Server.
- **Web of Trust**: To prevent spam or malicious actors from flooding the network with fake SOS alerts, Signal uses cryptographic signatures. Users can physically meet and "Vouch" for each other by exchanging public keys, creating a verified Web of Trust.
- **Real-Time Map & Feed**: Beacons are plotted on an interactive Leaflet map and listed in a live feed, categorized by urgency and type.
- **GPS Integration & Pin Drop**: Users can pinpoint exact danger zones using their device's built-in GPS or by dropping a pin directly on the map.
- **Voice-to-Text Support**: In high-stress situations, users can simply speak their emergency needs into the app.

---

## 📖 Terminologies

- **Beacon**: A single message broadcasted to the network. It has a type (SOS, Alert, Offer, Status), an urgency level, a location, and a cryptographic signature verifying who sent it.
- **Relay Bridge**: A local Node.js server that acts as a middleman for devices to sync with each other over a local network (e.g., a mobile hotspot) when the global internet is down.
- **Trust Edge**: A cryptographic proof that one device trusts another device. This forms the foundation of the Web of Trust.
- **Web of Trust**: A decentralized way to verify users. If you trust Person A, and Person A trusts Person B, the network calculates that Person B is somewhat trustworthy.
- **Public Key**: A unique, anonymous identifier generated by your device. You share this key to let others "Vouch" for you.

---

## 🛠️ How It Works (Technical Overview)

1. **Identity Generation**: On first load, the app generates a unique Elliptic Curve (ECDSA P-256) keypair using the WebCrypto API. This is your permanent identity.
2. **Data Storage**: All beacons and trust edges are stored locally in the browser using IndexedDB (via Dexie.js).
3. **Data Syncing (The Mesh)**: When two devices connect (either via P2P WebRTC or the Relay Bridge), they exchange a list of their local beacons. They compare timestamps and merge the newest data, effectively spreading information like a virus across the network.
4. **Signature Verification**: Every beacon is cryptographically signed by the sender's private key. When a device receives a new beacon during a sync, it verifies the signature before accepting it into its local database, ensuring the beacon was not tampered with.

## 📱 How to Use (User Guide)

### 1. Broadcasting a Signal
- Tap the **"+" (Plus) button** in the bottom right corner.
- Select the **Type** of beacon (e.g., Critical Need, Alert).
- Set the **Urgency**.
- Add a **Message** (you can type or use the Microphone button).
- In the **Location** section, tap **"📍 Use GPS"** to grab your current location, or **"🗺️ Drop Pin"** to manually select a spot on the map.
- Tap **Broadcast Signal**.

### 2. Syncing with Others (Mesh Networking)
To share your data with people around you:
- Tap the **🔗 Sync** icon at the bottom.
- Ensure you are on the same local network (e.g., connected to the same mobile hotspot).
- Click **"🌐 Sync via Relay"**. Your app will instantly pull all beacons from the network and push yours out.

### 3. Building the Web of Trust
To prevent fake alerts, verify the people you physically meet:
- Go to the **Trust** tab.
- Tap **"📋 Copy My Key"** and send it to your peer (via SMS, Bluetooth, etc).
- Your peer pastes your key into the **"Vouch for Peer"** box and taps **Vouch**.
- The Trust Graph will instantly update to show a physical connection between your devices.

## 💻 Installation & Setup (For Judges / Developers)

### Prerequisites
- Node.js installed on your machine.

### Running the App Locally

1. **Install Dependencies**
   Navigate to the `nexus` directory and install the frontend dependencies:
   ```bash
   cd nexus
   npm install
   ```
2. **Start the Relay Server**
   Open a new terminal, navigate to the relay server directory, install dependencies, and start it:
   ```bash
   cd nexus/relay-server
   npm install
   node index.js
   ```
   *The relay server will run on port 4000.*

3. **Start the Frontend Application**
   Back in your main `nexus` terminal, start the Vite development server and expose it to your local network:
   ```bash
   npm run dev -- --host
   ```
### Testing on Multiple Devices
To test the mesh network capabilities:
1. Connect your phone to your laptop's Wi-Fi network (or connect your laptop to your phone's mobile hotspot).
2. Look at your terminal output for the Network IP address (e.g., `http://192.168.1.5:5173`).
3. Open that IP address in your phone's browser.
4. Create a beacon on your phone, then go to the Sync tab and click "Sync via Relay". The beacon will instantly appear on your laptop!
